# Issue 2: 스와이프 카드 UI + 제스처 + 상태 관리 + 결과 화면

## Parent PRD

[PRD.md](../PRD.md)

## What to build

Issue 1에서 생성한 JSON 데이터를 사용하여 전체 스와이프 플로우를 구현한다. 틴더 스타일 카드 UI(음식 사진을 크게 표시, 식당명·카테고리 오버레이)를 만들고, 터치/마우스 드래그로 좋아요(오른쪽)/패스(왼쪽) 스와이프를 지원한다. 스와이프 상태를 관리하여 좋아요 1곳이 남으면 `/result`로 이동하고, 전부 패스하면 빈 상태를 표시한다. 결과 화면에서는 최종 식당 정보와 "다시 추천받기" 버튼을 제공한다.

## Acceptance criteria

- [ ] `/` 페이지에서 "교대" 지역 선택(고정) 후 `/swipe`로 이동할 수 있다
- [ ] JSON 데이터에서 음식점 목록을 읽어 랜덤 셔플하여 카드로 표시한다
- [ ] 카드에 음식점 이미지가 카드 면적의 대부분을 차지하고, 이름·카테고리가 오버레이로 표시된다
- [ ] 모바일에서 터치 드래그로 카드를 좌/우 스와이프할 수 있다
- [ ] 데스크톱에서 마우스 드래그 또는 하단 좋아요/패스 버튼으로 조작할 수 있다
- [ ] 드래그 중 카드가 이동 거리에 비례하여 회전한다 (최대 ±15도)
- [ ] 화면 너비 30% 미만 드래그 시 원위치로 스냅백된다
- [ ] 오른쪽 스와이프 시 초록색 오버레이 힌트, 왼쪽 시 빨간색 오버레이 힌트가 표시된다
- [ ] 남은 카드 수가 화면에 표시된다
- [ ] 좋아요한 식당이 1곳만 남으면 자동으로 `/result`로 이동한다
- [ ] 모든 카드를 패스하면 "추천할 식당이 없습니다. 다시 시도해주세요" 메시지와 재시작 버튼이 표시된다
- [ ] `/result` 페이지에 최종 식당의 이름, 카테고리, 이미지가 표시된다
- [ ] "다시 추천받기" 버튼 클릭 시 `/`로 이동한다
- [ ] `prefers-reduced-motion` 시 애니메이션이 줄어든다
